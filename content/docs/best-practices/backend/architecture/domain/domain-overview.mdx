## Entity Overview

The domain layer is organized by business/domain concepts, not by technical implementation details.

- Entities are the core of the domain layer.
- Guard classes are used to validate entities.
- Error classes are used to represent errors that can occur during the execution of the program.
- Value objects are used to represent value objects that are not entities.
- Refined types are used to represent refined types.




## Utilities Overview

### Base Entity (`base.entity.ts`)
Contains the `SimpleSerialized<T>` type utility for entity serialization.

### Refined Types (`refined.types.ts`)
Provides `Maybe<T>`, `normalizeMaybe`, and `optionToMaybe` utilities for handling optional values safely.
